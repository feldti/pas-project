Entwicklung - Neues Model laden

Einführung

PUM erzeugt eine neue Modellbeschreibung Ihrer Datenbank in der internen Sprache "topaz". Das ist ein Dialekt der Programmiersprache "Smalltalk", die angereichert ist mit Skript-Elementen. PUM erzeugt ein Shell-Skript, das ein topaz-Skript enthält.

Angenommen wird, daß Sie eine laufende Datenbank besitzen und die neue Modellversion in diese Datenbank importiert werden soll. Das geschieht über den Aufruf des folgenden Skriptes:

pas_load_topaz_source.sh <stone-name> <registry-name> <software-name> <branch> <version>

Wenn Sie unterschiedlichen Quellen unterstützen müssen, dann kopieren Sie sich das Skript und ändern in dem Skript die http-URL des Servers ab. Im Quelltext wird die Variable "BASEPATH" auf eine http-Adresse gesetzt_ "http://192.168.178.170/app-sources"

Auf dem Quellserver:

/var/www/html

           +  /app-sources

                    +     /surveymanager     ("Softwarename 'surveymanager'")

                                +       /v00           ("Branch v00")

                                          + /sources        

                                                +   /surveymanager-modelsource-0_001.sh   ("Spezielle Version 0_001")

So findet man also unter sources alle verschiedenen Modellversionen des Programmes als "topaz"-Skript. In der weiteren Entwicklung werden hier nicht nur die topaz-sh Skripte abgelegt, sondern auch die Monticello-Packages der Anwendung.

Beispiel für das Importieren des Modelles:

pas_load_topaz_source.sh devdb devregistry surveymanager v00 0_001
