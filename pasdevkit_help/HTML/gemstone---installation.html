Gemstone - Installation

Installation

Dieser Abschnitt beschreibt nur die Installation von Gemstone im Rahmen des PAS.

Zuerst muß einmal die Runtime-Umgebung für Gemstone installiert werden. Dazu lädt man sich ein Installationsskript herunter und führt es dort aus, wo der Ordner "pas" mit allen Daten angelegt werden soll.

Ich habe mir angewöhnt einen Root-Order anzulegen (/datadisk), und alle meine PAS- Anwendungen dort zu installieren. Dieser Root-Ordner sollte dem "normalen" Anwender gehören und am besten liegt das natürlich auf einer eigenen Partition - aber man kann auch das System im HOME-Verzeichnis des Benutzers installieren.

Also:

 

cd /datadisk

wget https://feldtmann.ddns.net/pas-project/pas_install_env.sh

chmod a+x pas_install_env.sh

./pas_install_env.sh

rm ./pas_install_env.sh

 

 

Danach liegt dort ein Ordner "pas"mit Unterordner "git" und zwei Projekten, die von github installiert wurde. Außerdem hat das Skript der lokalen ".bashrc" des Benutzers am Ende kleinere Einträge hinzugefügt:

 

export PAS_HOME_PATH=/datadisk/pas
export PAS_DEFAULT_REGISTRY=work
export PATH=$PAS_HOME_PATH/git/superDoit/bin:$PAS_HOME_PATH/git/GsDevKit_stones/bin:$PATH
export STONES_DATA_HOME=$PAS_HOME_PATH/registry

 

 

Einmal ausloggen und wieder anmelden, und die Skripte stehen zur Verfügung. Diese Einträge sind übrigens auch notwendig für die service-Skripte.

GsDevkit_stones

Das System basiert auf einer eigenen Version von GsDevKit_stones - die zusätzliche Beispiele und Skripte beinhaltet. Das führt dazu, daß in meinen Beispielen teilweise meinen eigenen Skripte, die aus dem Original von GsDevKit_stones und auch die Tools von Gemstone selbst genutzt werden. dabei muß man immer darauf achten, wann man welches Tool nutzen kann.

Der Vorteil von GsDevkit_stones liegt darin, daß die 32Bit Komponenten (Pharo basierend) entfernt wurden - also kein 32 Bit Subsystem mit all den Schwächen installiert werden muss, und verschiedene Anwendungen mit verschiedenen Lizenzen gestartet werden können. Es ist also möglich eine 3.7.5 Anwendung mit einer vollen Lizenz und eine andere 3.7.5 Anwendung auf einer eingeschränkten Version laufen zu lassen.
