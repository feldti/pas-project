Verbindung zur VM aufnehmen

SSH Verbindung zu den einzelnen VMs aufnehmen

Die VMs sind alle nur mit einem NAT als Netzwerk ausgerüstet. Man kann daher mit ssh oder Filezilla nicht direkt auf diese Maschinen Kontakt aufnehmen, aber man kann über den Host zur VM gelangen über den Befehl:

 

ssh -J <host-login>@host-ip <vm-login>@vm-ip 

 

 

Die VM-IP sieht so aus wie 192.168.122.x und der vm-login sollte per default immer "pas" heißen. Wenn man die IP Nummer nicht kennt, dann auf dem host folgendes ausführen:

 

virsh domifaddr <vm-name>

 

 

Damit das alles funktioniert, muß natürlich auf dem host openssh installiert sein:

 

# Status prüfen

sudo systemctl status ssh

 

# Falls nicht installiert

sudo apt install openssh-server

 

# Starten und dauerhaft aktivieren

sudo systemctl enable --now ssh

 

 

Falls es Probleme gibt, daß sich der Key des Hosts geändert hat, dann muß man auf seinem Arbeitsrechner

 

ssh-keygen -f '/home/mf/.ssh/known_hosts' -R host-ip

 

den Key entfernen. "mf" ist der Benutzer, von dem aus der ssh Befehl ausging.

Verbindung via Filezilla ermöglichen (noch nicht hinbekommen)

Filezilla versteht erst einmal keinen jump-host. Daher in ~/.ssh/config auf dem Arbeitsrechner eintragen:

 

Host <vm-rechnername>

    HostName vm-ip

    User vm-login

    ProxyJump host-login@host-ip

 

 

und in Filezilla eine Verbindung zu <vm-rechnername> herstellen.
