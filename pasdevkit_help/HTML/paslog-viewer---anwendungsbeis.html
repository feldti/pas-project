PASLOG-Viewer - Anwendungsbeispiel

Zielsetzung

Viel nachgedacht habe ich nicht, als ich diese Anwendungsentwicklung gestartet hatte. Am Anfang wurden folgendes angenommen:

•Eigenständige Anwendung in Gemstone/S mit UI Oberfläche in einer Universal UI von Sencha ExtJS 8.0.

•Die Anwendung wird auf dem Datenbankrechner installiert, wo die PostgreSQL Datenbank läuft

•Die Anwendung hat eigene Benutzerverwaltung. Ein Default-Admin wird mit der Anwendungsinitialisierung angelegt.

•Andere Anwendungen können diese LOG-Viewer Anwendung einfach in einem Frame mit erneutem Login einbinden oder auch nicht

•Die UI sollte anfangs für ein Browser geschrieben werden - aber die optionale UI für mobile Devices sollte möglich sein. Daher die Universal App.

•Als neue Anwendung sollte die neueste Version von Gemstone/S genutzt werden - also 3.7.5

oMinimale Datenhaltung in Gemstone/S. Benutzerverwaltung, Sessions, HTTP Traffic

oDatenhaltung primär in PostgreSQL

oAnwendung sollte mit der kleinsten, offenen Standardlizenz von Gemstone laufen können

•Bei PostgreSQL irgendetwas ab 16.0 und RabbitMQ ist sowieso immer dabei

oEs sollte Row-Level Filtering eingesetzt werden. Damit bestimmte Benutzer nur bestimmte LOG-Einträge sehen können (wie auch immer das gehen mag)

•PUM soll in diesem Projekt weiterentwickelt werden:

ofür ALLE CRUD Konstrukte sollen per default UIs erzeugt werden:

▪Panel mit Tabelle

▪Panel mit einer Form für das Domain-Objekt der Tabelle

▪Panel: kombiniert mit Tabelle und Form und einem Store

▪Window mit dem kombinierten Panel
