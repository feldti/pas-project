Und nun alles unter dem PI

Installation

So, und nun die ganze Installation unter einem Raspi PI5/8 GB.

Betriebssystem

Es wird das Image vom Juni 2026 genutzt mit Debian 13 / Trixie, installiert auf einer NVMe SSD. Die Installation von Trixie auf einer NVMe na ja, dass überlasse ich den zahlreichen Quellen im Internet. Ich habe mich nach dieser Quelle gerichtet, die aber auch schon veraltet ist. Nach Step9 habe ich rebootet (Das Mounten funktionierte in Step 10 nicht mehr) und das System startete von einer NVMe. Ich musste danach leider noch einmal die Tastatur konfigurieren und einen Benutzer angeben.

 

sudo apt update && sudo apt full-upgrade && sudo rpi-update

loginctl enable-linger

 

Apache2

Apache2 wird installiert:

 

sudo apt install apache2

 

Danach sollte man die Installation wie unter Apache2 beschrieben weiter verfolgen.

Wenn man eine dynamische Adresse hat, dann sollte man noch diese Seite berücksichtigen, um den noip-doc zu installieren .

PostgreSQL

Im Juni Image wird PostgreSQL 17 installiert via:

 

sudo apt install postgresql

 

git & subversion & ansible

 

sudo apt install git

sudo apt install subversion

sudo apt install ansible

 

openssh

Auch wenn er bereits installiert ist, solle man ihn auch aktivieren:

 

sudo raspi-config und dann über den Eintrag Interface Options

 

RabbitMQ

Mit der ausgelieferten Trixie Version wird RabbitMQ in der Version 4.0.5 ausgeliefert, ziemlich alt. Um neueren Code zu erhalten , sollte man hier nachschauen. Es wird einem auffallen, dass die Links nur für die AMD64 Technik zur Verfügung stehen, daher sollte man den folgenden Teil unter dem Reiter Trixie entsprechend austauschen und dann gelangt man an die Pakete für den PI.

 

## Add apt repositories maintained by Team RabbitMQ
sudo tee /etc/apt/sources.list.d/rabbitmq.list <<EOF
## Latest RabbitMQ releases
##
deb [arch=arm64 signed-by=/usr/share/keyrings/com.rabbitmq.team.gpg] https://deb1.rabbitmq.com/rabbitmq-server/debian/trixie trixie main
deb [arch=arm64 signed-by=/usr/share/keyrings/com.rabbitmq.team.gpg] https://deb2.rabbitmq.com/rabbitmq-server/debian/trixie trixie main
EOF

 

Mit diesen Änderungen führt man die Installation wie beschrieben auf der Seite durch.

.Net Core

Tja, für arm64-Architektur kann man erst bei .Net Core 10 anfangen: die Installation von Microsoft findet man hier.

Java

Mit dem Juni-Image wird die Version 21 von Java ausgeliefert:

 

sudo apt install default-jdk

 

Grafana/InfluxDB

Die Installation von Grafana findet man hier.

Prometheus

Bei der Installation habe ich die precompiled Variante ausgewählt. Der Download erfolgt über diese Seite. Die folgenden drei Programme sollten heruntergeladen werden:

 

wget https://github.com/prometheus/pushgateway/releases/download/v1.11.3/pushgateway-1.11.3.linux-arm64.tar.gz

wget https://github.com/prometheus/node_exporter/releases/download/v1.12.1/node_exporter-1.12.1.linux-arm64.tar.gz

wget https://github.com/prometheus/prometheus/releases/download/v3.13.1/prometheus-3.13.1.linux-arm64.tar.gz

 

Das pushgateway wird benötigt, wenn die Daten von einem entfernten Rechner nicht abgerufen werden können, sondern nur angeliefert werden können.

prometheus selbst ist der Server und die Datenhaltung.

Der node_exporter sammelt Daten für die lokale Maschine. Die Konfiguration des prometheus muß um diese Quelle erweitert werden:

 

scrape_configs:

  - job_name: "node_local"

    static_configs:

      - targets: ["localhost:9100"]
