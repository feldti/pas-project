PAS

Einführung

Mit PAS installierst Du die Verwaltungsumgebung von Gemstone/S Datenbanken. Auch wenn wir das eher in den VMs nutzen wollen, liefert es noch Hilfsroutinen für die weitere Installation der Komponenten auf dem Host.

Installation Host

Ich gehe in dieser Dokumentation immer davon aus, daß das "pas"-System in einem HOST unter "/datadisk" installiert wird. Das hat organisatorische Vorteile.

Wenn "/datadisk" ein symbolischer Link auf ein Verzeichnis im HOME eines Benutzers (z.B: Benutzer "pas"), dann sollte man darauf achten, daß das HOME-Verzeichnis die notwendigen Rechte hat - public readable ist - denn immerhin müssen die VMs in den shared-Bereich schreiben können.

Mit den folgenden Befehlen wird pas installiert (ich gehe immer davon aus, dass es unter "/datadisk" installiert wird):

 

cd /datadisk

wget https://feldtmann.ddns.net/pas-project/pas_install_env.sh

chmod a+x pas_install_env.sh

./pas_install_env.sh

 

Nach der Installation sollte man sich kurz abmelden ud wieder anmelden, damit die Änderungen in der Benutzersitzung ankommen.

Installation VM

Bei der Installation in einer VM kann das "pas" System im HOME-Verzeichnis des Benutzers "pas" durchgeführt werden. Ein "/datadisk" ist nicht erforderlich:

 

cd 

wget https://feldtmann.ddns.net/pas-project/pas_install_env.sh

chmod a+x pas_install_env.sh

./pas_install_env.sh

 

Installation Apache, PostgreSQL und RabbitMQ

Unter "pas/git/GsDevKits_stones/host-installation-helper" findet man eine Installationsroutine "pas_host_install.sh", die diese Komponenten installiert. Dabei werden (sollten) Unterschiede bei Ubuntu 22.04, 24.04 und Trixie berücksichtigt.
