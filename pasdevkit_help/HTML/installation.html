Installation

Vorbedingungen

Die folgenden Vorbedingungen sollten erfüllt sein:

•Nur KVM sollte als Virtualisierung genutzt werden. Keine parallele Installation von VirtualBox oder so.

Ziel

Die Ziele sind:

•"pas" und eventuelle Pakete der Anwendung (z.B: start/stop Skripte) sind in der VM installiert

•In der VM läuft der Datenbank-Kernel

•Die VM ist mit maximal 4 Cores ausgestattet und mit einer entsprechenden geringeren RAM Bestückung.

•Die Datenbanken speichern ihren "extent" im Host-Dateisystem und ebenso die Transaktionslogs.

•RabbitMQ und PostgreSQL laufen im Host (wenn nicht sogar auf einem anderen Rechner)

•Apache läuft im Host und leitet die Requests an die VMs weiter

Arbeitsschritte

Die Arbeitssschritte der Installation beginnen auf Seite 33 der Referenz.

Zuerst einmal werden die notwendigen Pakete installiert:

 

sudo apt update

sudo apt upgrade -y

sudo apt install -y qemu-kvm libvirt-daemon-system libvirt-clients bridge-utils virtinst virt-manager

sudo apt install -y virt-viewer virt-top libguestfs-tools guestfsd libosinfo-bin cpu-checker virtiofsd

sudo apt install -y cloud-image-utils

 

Starte den libvirt-Dienst

 

sudo systemctl enable --now libvirtd

sudo systemctl start libvirtd

 

Aktueller Benutzer der libvirtd Gruppe hinzufügen

 

sudo usermod -aG libvirt $USER

sudo usermod -aG kvm $USER

 

Neustart durchführen und weitermachen.

Test, ob Installation ok

 

kvm-ok (page 37)

virsh list --all (zeigt leere Liste an)

virsh net-list --all (zeigt defauöt metwork an)

virsh --connect=qemu:///system version (zeigt Informationen an)
